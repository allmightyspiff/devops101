Bluemix Overview

Think of Bluemix as a large collection of fully managed services that can be used on their own, or tied to Compute resources that run a customers application in Cloud Foundry or Docker containers on the Bluemix PaaS. It is somewhat comparable to the giant umbrella of products that are offered at AWS/Azure. While Bluemix doesn't always have the exact replacement for a specific AWS/Azure service, it will usually have a comparable product. In the Helpful Links section you can see a comparison chart titled 'IBM Bluemix for AWS professionals'.

Bluemix Pillars

IBM Bluemix is split in to 3 distinct pillars. These are Applications, Services, and Infrastructure. Most folks reading this will already have a good grasp on the infrastructure side of the Bluemix catalog (formerly SoftLayer), but may not be overly familar with the Bluemix PaaS offerings.

Applications

The application pillar of Bluemix includes all the options customers have to quickly get there code or service up and running on Bluemix. For most customers this will mean they are bringing some existing code or programming to Bluemix and not necessarily building something in Bluemix from the ground up.The applications pillar includes Cloud Foundry, Openwhisk, IBM Containers, and Mobile development services.

Services

Services are managed applications that can be added to a customers environment. Bluemix will provide account credentials to interact with the service that allows the user's application to take advantage of the service. These services include things like Watson, Data and Analytics Services, DevOps Offerings, Application Services.
